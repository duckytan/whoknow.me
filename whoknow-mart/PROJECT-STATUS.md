## 🧠 博弈引擎设计（7-21 錡哥定稿 · 核心特色）

### 前端（线上 · ¥0 运行成本）

**选招制** — 不是自由输入，也不是猜拳：

```
导购："这个你也买？上月房租交了吗？"

  ┌─────────────────────────────┐
  │  选择怎么反驳：               │
  │                              │
  │  ① 💪 我需要！（坚定眼神）   │
  │  ② 📊 我比过价了（理性证据） │
  │  ③ 🥺 求求了（装可怜）       │
  │  ④ 🤷 爱卖不卖（以毒攻毒）   │
  └─────────────────────────────┘
```

**机制**：
- 每个导购性格有隐藏弱点 → 选对招好感度 +40，选错 -10 或被喷
- 好感度攒满 → 导购叹气 → 下单成功
- 好感度归零 → "省钱了，不用谢我"
- 每轮出4个选项，位置随机，防止肌肉记忆

### 后台（7-21 錡哥定稿 · AI 驱动进化）

**核心思想**：AI 不当前台小丑，当**幕后游戏主**。

```
每天玩家对战记录 → 日志
                    ↓
         AI 引擎分析日志（每天跑1次 / 甚至每1小时1次）
                    ↓
    - 发现"毒舌导购被80%玩家破解"
    - 发现"0%人选第④个选项"
    - 发现"玩家在第3轮后胜率骤降"
                    ↓
    AI 自动生成新招式 / 调整克制关系
                    ↓
          线上配置更新（前端零成本）
```

| 维度 | 设计 |
|------|------|
| **跑AI时机** | 每天半夜定时跑（用户可配：1小时1次也负担得起）|
| **费用** | 极低（一次分析 = 几毛钱） |
| **核心逻辑** | AI 发现玩家破解模式 → 给导购换新招 |
| **效果** | 游戏深度持续进化，玩家永不破解到底 |

**对比传统游戏**：
```
传统手游：运营团队人工调平衡 → 慢、贵、不持续
胡闹导购：AI 自动发现套路 → 自闭环进化 → 每天更新
```

### 🔥 时事热点联动（7-21 錡哥拍板 · 双产品共享能力）

**核心思路**：AI 每天读热搜 → 自动注入两款产品，让用户「今天打开看到的是今天的段子」。

#### 对胡闹导购的应用：

| 热点类型 | 导购反应 | 效果 |
|---------|---------|------|
| 购物节（双11/618）| 导购火力全开："今天人傻钱多，我就不凑热闹了" | 用户逛淘宝时刚好撞上胡闹导购→截图对比 |
| 明星代言翻车 | 导购："这个明星昨天塌房了你还买？" | 当天热搜配当天吐槽 |
| 物价/经济新闻 | 导购："看到猪肉涨到多少了吗？你还买这个？" | 现实比段子更荒诞 |
| 科技新闻 | 导购："XXX刚发布新品你就买这个淘汰货？" | 装懂王 |
| 社会热点 | 商品临时改名联动 → 导购吐槽 | 今日限定梗 |

**例子**：
- 双11当天 → 导购开场白自动切换："今天？我劝你别买，过两天半价"
- 某明星塌房 → 导购选招选项新增："他不塌我不塌，我买我的你别叭叭"
- 高温红色预警 → 商品评论区 AI 注水："买家：这快递拿到手都化了"

#### 与胡闹外卖共享配置

```
同一套 AI 后台 → 输出两份配置：
  ├── whoknow-waimai.npc.json（老板吐槽 + 骑手剧情 + 菜品热点）
  └── whoknow-mart.npc.json（导购话术 + 商品热点 + 选项调整）

成本：1 次AI调用 → 同时更新两个产品
```
