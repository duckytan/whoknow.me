## 🔄 工作流程

### 定时任务（cron）

```
每日 03:00 触发
  │
  ├── 1️⃣ 采集层抓取
  │     ├── 微博热搜 Top 10
  │     ├── 今日新闻摘要（RSS）
  │     ├── 今日天气
  │     ├── 今日节日/纪念日
  │     └── 各产品昨日运营数据
  │
  ├── 2️⃣ AI 分析层调用 LLM
  │     ├── 输入：热搜 + 新闻 + 天气 + 节日
  │     ├── 输出：
  │     │   ├── whoknow-food  NPC 话术更新
  │     │   ├── whoknow-food  老板吐槽热点匹配
  │     │   ├── whoknow-food  骑手戏精台词更新
  │     │   ├── whoknow-food  戏精评价模板更新
  │     │   └── whoknow-food  彩蛋话术更新
  │     │
  │     ├── whoknow-mart 导购话术更新（后期）
  │     └── whoknow-mart 反骨选项平衡（后期）
  │     │
  │     └── 成本：1 次 LLM 调用，生成全部产品配置
  │
  └── 3️⃣ 分发层
        ├── 写入配置 JSON（版本号递增）
        ├── 上传 CDN / Git 仓库
        └── 生效时间标记（产品端按时间拉取）
```

### 前端拉取流程

```
产品启动 / 定时轮询
     │
     ├── 请求 GET https://cdn.whoknow.me/food/config.json
     │
     ├── ✅ 成功：
     │     ├── 校验 version 是否 > 本地缓存
     │     ├── 是新版 → 替换缓存 + 更新渲染
     │     └── 是旧版 → 继续用缓存
     │
     └── ❌ 失败（CDN 宕机 / 网络问题）：
           └── 继续用上一次缓存的配置 → 降级无感
```

## 🚨 降级策略

| 故障场景 | 前端行为 | 用户感知 |
|:--------:|----------|:--------:|
| CDN 挂了 | 使用上一次缓存的配置 | 话题落后一天，但功能正常 |
| brain 宕机 | 同 CDN 挂了 | ❌ 无感知 |
| 前端第一次启动 + 网络不通 | 使用内置 fallback 配置（预制模板）| 段子不变，但能跑 |
| AI 生成异常 | 使用上一次生成的配置 + 跳过一天 | 话题不变，¥0 损失 |
