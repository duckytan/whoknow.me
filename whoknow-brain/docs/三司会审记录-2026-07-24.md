## 🎯 第一轮（R1）：框架选型 + 多服务器架构

### 议题

- OpenClaw vs n8n vs crontab vs LangGraph
- 多服务器架构（M0 → M5）
- M0 范围
- 反馈机制

### 关键洞察

**最大转折点**：从"框架选型"重新对齐到"业务驱动"。
- 锡哥原话："底层架构用什么比较好？是 openclaw？还是 harmars？"
- 揭示出：锡哥想"另开服务器"，重新架构 brain
- 结论：M0 用 OpenClaw（已有栈），不另起炉灶

### R1 共识（7 个）

1. ✅ **brain 不只是内容生成，还管服务器数据 + 玩家反馈**
2. ✅ **反馈按钮用 👍/👎 二表情**（三表情无意义，"中间那个表情，意义何在？我不操作，默认也是中间"）
3. ✅ **M0 不通知锡哥，锡哥去 brain 后台看**
4. ✅ **服务器数据"应用本身记录"= 浏览器端埋点**（不是后端服务器监控，那是 M1）
5. ✅ **M0 范围 = 老子平衡 12 项清单**
6. ✅ **JSON schema 不可逆，先定再写代码**
7. ✅ **M0 手动触发，不挂 cron**

### R1 警示（5 个）

| # | 警示 | 严重性 |

- AI 话术质量保证

### R2 共识（7 个）

1. ✅ **大矩阵**（未来 5+ 产品，brain 是中枢）
2. ✅ **brain 服务全部 + 以后所有**
3. ✅ **NPC 少而精**（先 5-8 个，质量优先）
4. ✅ **M0 验收看心情，但 M0 阶段至少每天 1 次**
5. ✅ **审核渐进迁移**（人工 → 智能，AI 学习）
6. ✅ **7 感官 M0 我决定**（含后端 + 前端埋点）
7. ✅ **成本 ¥30/月**

## 👥 第三轮（R3）：4 角色视角

### 议题

- 用户视角（22 岁独居 Z 世代）
- 投资者视角（早期投资人）
- 竞争者视角（美团 PM）
- UP 主视角（B 站 100w 粉 UP）

### 关键洞察

**最大转折点**：识别 brain 的"客户"分层。

| 角色 | 优先级 | 需求 |
|------|:----:|------|
| **用户** | 🥇 第一 | 好玩 / 不累 |
| **UP 主** | 🥈 第二 | 3 秒爆点 + AI 故事 |
| **投资人** | 🥉 第三 | 留存 / 壁垒 / 可量化 |
| **竞争者** | 防御 | NPC 人格 IP |

### R3 共识（5 个）

1. ✅ **brain 第一客户是用户**（不是投资人）
2. ✅ **brain 的产品是"用户每天看到的体验 + 锡哥每周节省的时间"**
3. ✅ **brain 护城河是 NPC 人格 IP + 玩家行为数据 + 段子文化**（不是 prompt）
4. ✅ **brain 最大风险是"段子质量下降 → 玩家流失 → 锡哥疲惫"**
5. ✅ **M0 验证 3 个核心**：话术质量、节省时间、AI 故事真实性

### R3 警示（8 个）

| # | 警示 | 严重性 | 来源 |
|---|------|:----:|------|
| 1 | 前 3 秒决定生死 | 🔴 P0 | 用户 |
| 2 | MVP 阶段投资人会 PASS，先别想融资 | 🔴 P0 | 投资者 |
| 3 | 同质化是真实威胁，必须 NPC 人格独立化 | 🔴 P0 | 竞争者 |
| 4 | "AI 每天更新"是传播点，UI 要有水印 | 🟡 P1 | UP 主 |
| 5 | 段子失信 = 传播反噬，AI 一定要真接热搜 | 🔴 P0 | UP 主 |
| 6 | 护城河不是 prompt，是 NPC 人格 IP | 🔴 P0 | 竞争者 |
| 7 | 锡哥 1 人扛 1 年扛不动 = 项目死 | 🔴 P0 | 投资者 |
| 8 | 反馈按钮必须"顺手"，否则没人用 | 🟡 P1 | 用户 |

### R3 关键洞察：brain 是运营代理人

> 锡哥 Q27 原话：
> "才省 1 小时？有了 brain 后，数据上的问题，brain 就必须全盘接管，我全力去开发新 app。我只能偶尔抽样看。所以 brain 非常重要，是整个胡闹宇宙的基石。"

**结论**：brain 不能出问题。brain 必须能独立运转。
