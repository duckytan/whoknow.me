## 决策 #013：胡闹事件彩蛋机制
**日期**：2026-07-22
**决策**：B 方案 — 20% 订单触发胡闹彩蛋，100% 送达

**背景**：锡哥提出「买家大概率吃不上外卖」的剧情方向（店家睡着/骑手被抓/食物变异）

**讨论记录**：
- 🟢 我方反对「永远吃不上」的理由：
  1. 商业模式直接死亡（0 单 = 0 收入）
  2. 胡闹导购/大脑的品调也需同步改动
  3. 虚拟用户（想体验但减肥的人群）流失
- 🟢 方案 B（20% 彩蛋 + 100% 送达）平衡了胡闹感与商业可行性
- 🟢 段子附赠的 slogan 已足够支撑「胡闹」调性

**核心规则**：
1. 80% 订单 = 正常配送 + 附赠段子（维持原设计）
2. 20% 订单 = 触发随机彩蛋事件，但 100% 送达
3. 彩蛋类型：骑手讲相声/餐盒多图/店铺送明信片/机器人电话讲段子等
4. 不破坏「点得倒外卖」的体验承诺

**暂不实施**（当前只做品牌页，无真实下单流程，等 MVP 阶段再开发此机制）
**拍板人**：锡哥（选 B）

## 决策 #014：Onboarding 引导文案（v14 P0-5 · 7-23 拍板）

**决策**：新用户首次进站弹 **3 屏引导**，回答 3 个问题（是什么 / 怎么玩 / 彩蛋在哪）

**背景**：M1/M2 已完成，但 P0-5 onboarding 文案一直空着。用户首次进站可能直接被轮播图和分类 icon 迷惑，不知道「这是假外卖」。

**3 屏内容**：

| 屏 | 标题 | 描述 |
|----|------|------|
| 1 | 🎭 假外卖，真段子 | 这里没有真饭。只有 15 个戏精老板和 5 个奇葩骑手，陪你演一出好戏。 |
| 2 | 🍜 随便点，没成本 | 想点啥点啥，0 元下单不给真吃。放心吐槽、放心给差评，他们都会接住。 |
| 3 | 🥚 彩蛋藏在这 | 备注栏写点奇葩的 / 地址选"百慕大" / 给 5 星时手抖——NPC 会给你专属演出。 |

**触发逻辑**：
- Home.vue `onMounted` 检查 `localStorage.chaos_onboarded`
- 未标记 → 弹 → 末屏"开始玩 🎭" → 写 `1` + 关闭
- 已标记 → 不弹（不打扰）

**设计原则**：
- 30 秒可读完（3 屏 × ≤10 秒）
- 文案有"哇"的瞬间，不是说明书
- 末屏"开始玩"按钮永远在底部
- `van-popup position="bottom"` + `teleport="body"` + 全屏 100vh

**改动文件**：
- 新增 `src/components/base/OnboardingGuide.vue`（4485 字节）
- 改 `src/pages/Home.vue`（加组件 + onMounted 检查）
- 改 `src/utils/storageKeys.ts`（新增 `onboarded: 'chaos_onboarded'`）

**验证**：
- vue-tsc 0 错
- vite build 1.25s
- dist Home bundle 9.97 kB（v12 是 7.63 kB，+2.3 kB 可接受）

## 决策 #021：M2 闭环完善（v15 · 7-23 拍板）

**决策**：补 M2 三个最值得做的体验缺口 —— **评价提交庆祝态 + 分享按钮 + 未评价订单红色徽标 + AI 引导提示**

**背景**：M1+M2 基础已完整，但用户视角走「下单 → 收货 → 评价 → 历史」流程仍有 3 个体验断层：

| 缺口 | 影响 | 决策 |
|------|------|------|
| 评价提交后无反馈 | 用户不知道成功了，**无法触发传播** | 加庆祝态（动画 + 分享按钮） |
| OrderHistory 没催评 | 完成订单但没评价，**30% 用户漏评** | 加红色脉冲徽标 |
| ReviewModal 进入无引导 | 用户看到一堆字段困惑 | 加 AI 引导"已为你写好初稿" |

**核心设计原则**（SOUL 产品哲学铁律）：
- **用户能传播吗？** → 必须有截图分享按钮（navigator.share + clipboard 降级）
- **用户会爽吗？** → 提交后"好戏已发布"动画 + 老板回复预览
- **用户能用吗？** → AI 已写好初稿，点一下直接编辑

**改动文件**：
- 改 `src/components/review/ReviewModal.vue`（加 `submitted` 状态、庆祝模板、分享逻辑、AI 引导）
- 改 `src/pages/OrderHistory.vue`（加 `review-pending` 红色脉冲徽标）

**技术亮点**：
- `navigator.share` 优先（H5 / 移动 Web）；不支持降级 `navigator.clipboard.writeText` → 再降级 `document.execCommand('copy')`
- 三档 toast 自实现（不依赖 Vant import，避免循环依赖）
- 分享文本格式：`【胡闹外卖】给 XXX 打了个 X 星：「...」老板回复：... #胡闹外卖`

**验证**：
- vue-tsc 0 错
- vite build 1.31s
- OrderHistory bundle 4K CSS + 4K JS（增量小）

**拍板人**：锡哥（7-23 08:34 拍"继续 A"）
