## 🎯 审查锚点（产品哲学铁律 · 锡哥金句）

> **「你要学会从用户的思维去想问题」**
>
> 任何产品决策前，必须能回答 3 个问题：
> 1. **用户能用吗？** — 30 秒内能学会吗？
> 2. **用户会爽吗？** — 用完会有"哇"的瞬间吗？
> 3. **用户会传播吗？** — 会想截图/发朋友圈/告诉朋友吗？

**3 个都不能清晰回答 → 不做这个功能**

## 🌳 木 · 根（太初诀 · 第一性）

### 拆到不能再拆：MVP 的"最小用户闭环"是什么？

**事实卡片**（决策 #004 Phase 1）：
```
用户打开 App → 浏览 20 NPC 商家 → 加入购物车 → 下单
→ NPC 商家接单（2-5 分钟）→ NPC 骑手抢单（5-10 分钟）
→ 地图移动 SVG → 送达 → 写戏精评价 → 获得虚拟币
```

### 三个用户视角问题

#### Q1：用户能用吗？（30 秒能学会？）
**判定**：✅ **能用**
- 美团 UI 1:1 = 用户**已经在用**——打开就有"哦我懂"的肌肉记忆
- 30 秒学习成本 = 0
- **风险**：但 1:1 复刻 ≠ 真美团——Vant UI 和真实美团有差异，用户可能困惑"这是美团新版本吗？"
- **改进**：第一次进入加 onboarding 提示"这是游戏不是真外卖"

#### Q2：用户会爽吗？（"哇"的瞬间在哪？）
**判定**：⚠️ **有风险**

**5 个可能的"哇"瞬间**：
| 时刻 | 类型 | 强度 |
|------|------|------|
| NPC 商家接单时的"老板甩话" | 😄 笑 | ⭐⭐⭐ |
| NPC 骑手地图 SVG 动画移动 | 👀 视觉 | ⭐⭐⭐⭐ |
| 戏精评价系统可玩 | 🎭 共创 | ⭐⭐⭐ |
| 虚拟币"看广告得金币" | 💰 实惠 | ⭐⭐ |
| 三 Tab 切换（看 NPC 老板/骑手视角） | 🎬 角色扮演 | ⭐⭐⭐⭐⭐ |

**关键发现**：最强的"哇"瞬间是**三 Tab 切换**——但决策文档只把它列为"演示"，没突出为**核心卖点**。

#### Q3：用户会传播吗？（会想截图发朋友圈吗？）
**判定**：✅ **会传播**

**3 个截图驱动点**：
- NPC 老板甩话（"你的订单我看到了，但我心情不好，5 分钟后再来"）
- 戏精评价（"这小龙虾好吃到我愿意把手机吃掉"）
- 地图上骑手骑乌龟 SVG（视觉反差）

**风险**：截图后**不带产品水印**= 用户想传播但找不到 App 名 = 流量浪费
- **改进**：截图强制加水印（"胡闹外卖"logo + 二维码）

## 🔥 火 · 蔓（观复诀 · 二阶推演）

### Phase 1 上线 1 周后，会发生什么？

#### 场景 A：用户真的来玩了
- **用户行为**：打开 → 浏览 5 个 NPC 商家 → 选老王烧烤 → 加 3 个菜 → 下单
- **2 分钟内看到**："老王"甩话"来了来了别催，再催我关店"
- **5 分钟**：骑手"张师傅"抢单 → 地图上骑电动车 SVG 移动
- **用户反应**：截图 → 发小红书"救命这个 App 太沙雕了"
- **1 天后**：用户回来 → 发现订单状态还在（localStorage）→ 再玩

#### 场景 B：用户打开 30 秒后走了（最大概率）
- **用户行为**：打开 → 看到 20 个 NPC 商家 → 疑惑"这能下单吗？"
- **30 秒内没有引导** → 关闭
- **根本原因**：**没有"立即开玩"的钩子**——首页就是商家列表，没人告诉他"赶紧下单看戏精"

#### 场景 C：用户下单后没写评价（80% 概率）
- 戏精评价被设计成"主动写"而非"被动推"
- 50 条模板用户 1 周内看穿
- **改进**：NPC 主动 @用户："大哥你倒是评价啊，我等着上热搜呢"

### 关键推论
**Phase 1 的"传播力"不取决于功能多，而取决于"前 30 秒钩子 + 戏精评价的惊喜度"**——这是单点爆破，不是全面铺开。

## ⛰️ 土 · 容（洞观诀 · 系统回路）

### 这事在哪个大系统里？

```
        中国 Z 世代情绪消费系统
                ↑
    ┌───────────┼───────────┐
    │           │           │
 抖音"沙雕"    小红书      多巴胺 App
 段子          反差图      （FoodNeverComes）
    │           │           │
    └───────────┼───────────┘
                ↓
          胡闹外卖（情绪仪式 + 多角色）
                ↓
        截图 → 小红书/抖音 → 引流回 App
```

### 关键反馈环
- **正环 A**：用户发截图 → 小红书引流 → 新用户下载 → 写戏精评价 → 发截图
- **正环 B**：NPC 老板甩话很沙雕 → 用户截图 → 引流
- **断环 C**：用户换手机 → localStorage 清零 → **新用户不是老用户**，留存断裂

### 系统洞见
**这是个"内容驱动 + 仪式驱动"的产品，不是"工具驱动"**——抖音/小红书是它的销售渠道，不是竞争对手。

## ⚔️ 金 · 断（颠倒诀 · 反者道之动）⭐

### 怎么保证它失败？（顺则凡逆则仙）

**10 种必败模式**：

| # | 失败模式 | 触发条件 | 概率 |
|---|---------|---------|------|
| 1 | **"我看不出这是游戏"** | UI 太像真美团 + 没 onboarding | 🔴 高 |
| 2 | **"下单了但没人理我"** | NPC 接单逻辑太死板，重复同一句话 | 🔴 高 |
| 3 | **"老板甩话好无聊"** | 50 条模板 1 周看穿 | 🔴 高 |
| 4 | **"地图上骑手卡住了"** | SVG 动画 bug，骑手骑到墙里 | 🟡 中 |
| 5 | **"换手机就清零"** | localStorage 局限 | 🟡 中 |
| 6 | **"广告看半天给 30 金币"** | 虚拟币经济崩盘 | 🟢 低（Phase 1 量小） |
| 7 | **"跟朋友说我玩这个，Ta 问哪里下"** | 没分享按钮 / 截图没水印 | 🔴 高 |
| 8 | **"卸载了，没理由装回来"** | 没有推送 / 没有每日任务 | 🟡 中 |
| 9 | **"应用商店搜不到"** | 名字太普通 / 上架审核被拒 | 🟡 中 |
| 10 | **"老板甩话冒犯我了"** | 文案没审核（暴粗/歧视） | 🟡 中 |

### 怎么避开？
1. 首次进入 onboarding 3 句话："这是游戏 · 这是单设备 · 老板甩话是 NPC"
2. NPC 接单话术**模板化但变量化**（"来了"+"别催"+"饿了么不是我"+"我心情"+随机情绪）
3. 截图按钮放显眼位置 + 强制水印
4. SVG 动画写完后**录 5 个视频**自测，不允许出现骑手骑出地图

## 🌊 水 · 流（知几诀 · 带置信度）

### 我现在的先验把握

| 论断 | 置信度 | 改变所需证据 |
|------|--------|------------|
| 中国 Z 世代会买单 | 70% | 上线 1 周 DAU ≥ 100 |
| 戏精老板甩话是核心钩子 | 85% | 截图率 ≥ 5% |
| 三 Tab 切换是核心卖点 | 90% | 用户调研"哪个功能最想分享" |
| localStorage 够用 Phase 1 | 60% | 用户投诉"换手机清零" ≤ 10% |
| 12 天工期能完成 | 50% | 实操中会发现 Vant 主题色改造 ≥ 1 天 |
| 美团 1:1 复刻能过应用商店 | 40% | 应用商店审核反馈 |
| 用户会传播 | 80% | 截图率 ≥ 5% |

### 后验更新规则
- **上线 7 天**：DAU ≥ 100 → 置信度 +20%；DAU < 30 → 置信度 -40%
- **应用商店审核**：通过 → +30%；被拒 → 重做决策
- **截图率**：≥ 5% → 传播模型确认；< 1% → 戏精钩子失败

| 关系 | 哪 2 维 | 实质 |
|------|---------|------|
| **木生火**（顺势）| 太初 → 观复 | 拆完事实 → 推演二阶场景 |
| **火生土**（顺势）| 观复 → 洞观 | 推演后看清系统回路 |
| **土生金**（顺势）| 洞观 → 颠倒 | 看懂系统 → 找失败模式 |
| **金生水**（顺势）| 颠倒 → 知几 | 找完失败 → 带置信度决策 |
| **水生木**（顺势）| 知几 → 太初 | 更新把握 → 重新拆解 |

**5 维成闭环**——这是五行诀最理想的运行状态。

**8 关系检查**：
- 哪 2 维相克？**木克土**（太初拆太细会陷入细节，忽略系统）
- 有没有相乘？**金乘木**（颠倒诀太强可能让项目不敢启动）
- 有没有相侮？**水侮火**（知几带置信度可能过度乐观）

**平衡建议**：
- 木克土风险 → 决策层用五行诀时必须**先看洞观再拆太初**
- 金乘木风险 → 颠倒诀找的失败模式必须**有应对方案**，不能光找不做
- 水侮火风险 → 知几的置信度必须**有反向证据修正**，不能拍脑袋

## 🎯 用户体验审查结论

### 核心判断（按锡哥三问）

| 问题 | 答案 | 风险 |
|------|------|------|
| **用户能用吗？** | ✅ 能（30 秒） | onboarding 没写 |
| **用户会爽吗？** | ⚠️ 有风险（看 NPC 钩子） | 戏精话术 1 周被看穿 |
| **用户会传播吗？** | ✅ 会（截图驱动） | 没水印 + 没分享按钮 |

### 推荐行动（3 项 P0）

#### 🔴 P0 #1：写 onboarding 3 句话
- "这是游戏不是真外卖"
- "这是单设备游戏，换手机=重新开始"
- "老板甩话都是 NPC 别当真"

#### 🔴 P0 #2：截图按钮 + 强制水印
- 订单完成页 → 大大的"截图炫耀"按钮
- 截图自动加"胡闹外卖"logo + 二维码
- 微信/微博/小红书分享直达

#### 🟡 P1 #3：NPC 钩子升级
- 接单话术**变量化**（不是 50 条固定，而是 5 模板 × 5 变量 = 125 种组合）
- 增加"老板今日心情"随机维度（暴脾气/佛系/装死）
- 用户第 3 次下单解锁"老板记住你了"机制

### 总体判断

> **用户体验核心成立，但有 3 处"前 30 秒钩子"设计缺口**——这是 Phase 1 最大的 P0 风险，必须在 Day 1-3 设计期解决，不能等到测试期才发现。
