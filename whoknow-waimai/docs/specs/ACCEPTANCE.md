## Day 1 验收 · 项目骨架

**验收方式**：在浏览器打开 `localhost:5173`，对照以下清单

- [ ] 浏览器打开无报错（控制台 0 个红色 Error）
- [ ] 页面能正常渲染（不是空白页）
- [ ] 导航栏/底部 TabBar 可见
- [ ] 点击 TabBar 能切换页面（路由跳转正常）
- [ ] 主题色是**胡闹橙 `#ff6b35`**，不是默认蓝/绿
- [ ] 手机模拟器（Chrome DevTools → 手机图标）下 UI 不变形
- [ ] `vercel preview` 部署后可用链接访问

**不要求**：内容数据、美观度（Day 1 是骨架，允许空页面）

## Day 2 验收 · 首页 + 商家列表

**验收方式**：手机模拟器下操作

- [ ] 首页顶部可见 slogan「**零卡路里的外卖，越点越瘦**」
- [ ] 轮播图自动播放（最少 2 张）
- [ ] 分类 icon 区域可见（最少 6 个分类）
- [ ] 商家列表可见 **≥ 10 家**商家卡片（目标 15 家）
- [ ] 每张商家卡片包含：名字 + 评分 + 月销量 + 配送费 + 配送时长
- [ ] 点击商家卡片能跳转到商家详情页（页面存在即可，内容 Day 3 完善）
- [ ] 列表可以上下滚动（不被截断）
- [ ] 15 家商家中，5 种性格（angry/gentle/weird/lazy/philosophical）各至少 1 家

**不要求**：搜索功能、筛选排序（M2 功能）

## Day 3 验收 · 商家详情 + 购物车

**验收方式**：完整操作一次加购流程

- [ ] 进入商家详情页，能看到该商家的菜品（≥ 5 道）
- [ ] 左侧分类 + 右侧菜品「左右联动」：点左侧分类，右侧自动滚到对应位置
- [ ] 每道菜显示：菜名 + 价格 + 图片（或占位图）+ 加减按钮
- [ ] 点「+」按钮：菜品数量 +1，购物车总价更新
- [ ] 点「-」按钮：菜品数量 -1，减到 0 时「-」按钮消失
- [ ] 底部购物车浮层：显示**总价**和**商品总数量**
- [ ] 购物车浮层「去结算」按钮可点击，跳转到下单页
- [ ] 刷新页面后，购物车数据**保留**（localStorage 持久化）
- [ ] 100 道菜数据完整（`src/data/dishes.json`，`wc -l` 可验证行数）

**不要求**：菜品详情弹窗（可选功能）、规格选择

---

## Day 4 验收 · 下单流程 + NPC 状态机

**验收方式**：完整下一单，计时观察 NPC 行为

- [ ] 下单确认页显示：收货地址 + 商品列表 + 总价 + 备注输入框
- [ ] 点「确认下单」后：跳转到订单追踪页
- [ ] 订单追踪页显示当前状态（接单中/出餐中/配送中/已送达）
- [ ] **NPC 自动推进**（不需要手动操作）：
  - [ ] 下单后 2-5 秒：状态变为「商家已接单」，老板说话气泡出现
  - [ ] 5-10 秒后：状态变为「出餐中」
  - [ ] 再 1-3 秒：状态变为「骑手已接单」，骑手说话气泡出现
  - [ ] 10-20 秒后：状态变为「已送达」
- [ ] 老板气泡内容与商家性格匹配（angry 的店老板说话不温柔）
- [ ] 每次下单 NPC 说的话**不完全固定**（至少 2-3 种变化）
- [ ] 下单后购物车**自动清空**
- [ ] 订单状态持久化：刷新页面后订单还在，状态继续

**不要求**：地图动画（Day 5）、精确计时（允许 ±5 秒误差）

---

## Day 5 验收 · NPC 话术 + 骑手地图

**验收方式**：连续下 5 单，观察话术和动画

- [ ] `src/data/quotes.json` 存在，模板数量 ≥ 20 条
- [ ] 老板话术包含变量替换（能看到商家名字/菜品名出现在话术里）
- [ ] 5 种性格的老板话术风格明显不同（angry 说话凶，philosophical 说话文艺）
- [ ] 骑手说话气泡出现，内容与骑手座驾类型有关
- [ ] 骑手地图动画存在（SVG 小图标从起点移动到终点）
- [ ] 地图动画配合订单状态：「配送中」时骑手图标开始移动，「已送达」时停止
- [ ] **连续下 5 单**：NPC 话术不完全相同（允许偶尔重复，但不能 5 单完全一样）

**不要求**：真实地图（SVG 简版即可）、骑手路径弯曲（直线 OK）

## Day 6 验收 · 戏精评价 + 完整闭环

**验收方式**：走一遍完整下单到评价流程

- [ ] 订单「已送达」后，弹出彩蛋卡片（一句搞笑送达感言）
- [ ] 彩蛋卡片消失后或点击后，出现评价弹窗
- [ ] 评价弹窗包含：星级评分（1-5 星）+ 标签选择 + 评论文本
- [ ] 标签库包含胡闹系标签（「老板脾气好大」「骑手骑的像在飞」等 ≥ 6 个）
- [ ] 评论文本有预填内容（戏精评价，可编辑）
- [ ] 点「提交评价」后：弹窗关闭，商家详情页能看到这条评价
- [ ] 完整流程：首页 → 进店 → 加购 → 下单 → NPC 演戏 → 送达 → 评价，**全程无报错崩溃**
- [ ] 4 处品牌元素全部到位：① Logo ② 首页 Slogan ③ 评价戏精标签 ④ 送达彩蛋页

**不要求**：评价列表分页、评价统计（M2 功能）

- [ ] 首页首屏加载 < 5 秒（手机 4G 网络模拟）
