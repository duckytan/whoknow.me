## §1 项目骨架与基本体验

**验收方式**：浏览器打开 `https://whoknow.me/waimai/`，DevTools Console + Application + Network 面板辅助

### 基础渲染

- [ ] 浏览器打开无报错（DevTools Console **0 个红色 Error**）
- [ ] 页面能正常渲染（不是空白页 / 不是 loading 占位卡死）
- [ ] 导航栏（AppHeader）和底部 TabBar（AppTabBar）可见
- [ ] 点击 TabBar 能切换页面（路由跳转正常）
- [ ] 主题色是**胡闹橙 `#ff6b35`**，不是默认蓝/绿

### 移动端适配

- [ ] iPhone 12 Pro 模拟器（390px）：无横向滚动条，内容不溢出
- [ ] Android 常见尺寸（360px）：布局正常
- [ ] 所有可点击元素（按钮/卡片）手指可准确点击（**触摸目标 ≥ 44px**）

### 性能基准（dev/standard 修订）

- [ ] 首页首屏加载 **< 3 秒**（Chrome DevTools 4G 模拟）
- [ ] 路由切换 **< 1 秒**（点击 → 下页加载完成）
- [ ] 资源 Warning ≤ 3 个（图片 / 字体加载相关允许）
- [ ] 图片加载失败时，有占位图兜底（**不显示破碎图标**）

## §2 商家与购物车

**验收方式**：手机模拟器下操作一遍

### 首页（商家列表）

- [ ] 首页顶部可见 slogan「**零卡路里的外卖，越点越瘦**」
- [ ] 轮播图自动播放（≥ 2 张）
- [ ] 分类 icon 区域可见（≥ 6 个分类）
- [ ] 商家列表可见 **≥ 10 家**商家卡片（目标 15 家）
- [ ] 每张商家卡片包含：名字 + 评分 + 月销量 + 配送费 + 配送时长
- [ ] 列表可以上下滚动（不被截断）
- [ ] 15 家商家中，5 种性格（angry/gentle/weird/lazy/philosophical）各至少 1 家

### 商家详情

- [ ] 进入商家详情页，能看到该商家的菜品（≥ 5 道）
- [ ] 左侧分类 + 右侧菜品「左右联动」：点左侧分类，右侧自动滚到对应位置
- [ ] 每道菜显示：菜名 + 价格 + 图片（或占位图）+ 加减按钮
- [ ] 点「+」按钮：菜品数量 +1，购物车总价更新
- [ ] 点「-」按钮：菜品数量 -1，减到 0 时「-」按钮消失
- [ ] 菜品数据完整（`dishes.json` JSON 可解析 + 每条菜都有 name/price/category/id 字段）

### 购物车持久化

- [ ] 底部购物车浮层：显示**总价**和**商品总数量**
- [ ] 购物车浮层「去结算」按钮可点击，跳转到下单页
- [ ] 刷新页面后，购物车数据**保留**（localStorage 持久化）

## §3 下单与 NPC 状态机

**验收方式**：完整下一单，观察 NPC 5 阶段演出（demo 30× 默认）

### 下单流程

- [ ] 下单确认页显示：收货地址 + 商品列表 + 总价 + 备注输入框
- [ ] 点「确认下单」后：跳转到订单追踪页
- [ ] 下单后购物车**自动清空**
- [ ] 订单状态持久化：刷新页面后订单还在，状态继续

### NPC 自动推进（30× demo）

- [ ] 下单后 ~1-3 秒：状态变为「商家已接单」，老板说话气泡出现
- [ ] 3-10 秒后：状态变为「出餐中」
- [ ] 再 1-3 秒：状态变为「骑手已接单」，骑手说话气泡出现
- [ ] 15-30 秒后：状态变为「已送达」

### NPC 话术多样性与性格匹配

- [ ] 老板气泡内容与商家性格匹配（angry 的店老板说话不温柔）
- [ ] 每次下单 NPC 说的话**不完全固定**（至少 3-5 种变化）
- [ ] 5 种性格的老板话术风格明显不同（angry 说话凶，philosophical 说话文艺）
- [ ] 骑手说话气泡出现，内容与骑手座驾类型有关
- [ ] **连续下 10 单**：老板话术 + 评价标签 + 评价正文 3 项中至少 2 项每次都不同

## §4 评价与历史订单

**验收方式**：完整下 1 单 → 收货 → 写评价 → 看历史

### 评价流程

- [ ] 订单「已送达」后，弹出彩蛋卡片（一句搞笑送达感言）
- [ ] 彩蛋卡片消失后或点击后，出现评价弹窗
- [ ] 评价弹窗包含：星级评分（1-5 星）+ 标签选择 + 评论文本
- [ ] 标签库包含胡闹系标签（**≥ 6 个**，如「老板脾气好大」「骑手骑的像在飞」等）
- [ ] 评论文本有预填内容（戏精评价，可编辑）
- [ ] 点「提交评价」后：弹窗关闭，订单详情页能看到这条评价

### 历史订单

- [ ] 历史订单页可见所有下过的单（`/orders` 路由）
- [ ] 首页 TabBar 有快捷入口「我的订单」
- [ ] 已评价订单显示绿色「📝 已评价」徽标

### 完整闭环

- [ ] **完整流程**：首页 → 进店 → 加购 → 下单 → NPC 演戏 → 送达 → 评价，**全程无报错崩溃**

## §5 合规三件套（v16 新增 · 个人开发者合规）

**验收方式**：检查 3 个合规组件的实际内容与跳转

### 顶部合规 banner

- [ ] 顶部固定 banner 可见（半透明 + 胡闹橙主题色）
- [ ] banner 初始态：「虚构作品 · 不提供真实餐饮服务」（缩略文案）
- [ ] 点击 banner 展开后：「本应用为虚构娱乐作品，不提供真实餐饮服务」+ 「隐私政策」「用户协议」链接

### 隐私政策页

- [ ] 顶部 banner 链接「隐私政策」可跳转到 `/waimai/privacy`
- [ ] 隐私政策页包含 4 段：① 一句话总结 ② 这是个啥 ③ 我们存了什么 ④ 我们没做的事
- [ ] 隐私政策明确「不收集个人信息」「localStorage 存储」「不接第三方追踪 SDK」

### 用户协议页

- [ ] 顶部 banner 链接「用户协议」可跳转到 `/waimai/terms`
- [ ] 用户协议页包含 5 段：① 这是个啥 ② 你可以做的 ③ 你不能做的 ④ 免责声明 ⑤ 变更
- [ ] 用户协议明确「虚构作品」「0 元」「不是真外卖」「适用中国法律」

### 合规约束

- [ ] 无第三方追踪 SDK 代码（grep 无 GA / 百度统计 / Sentry 等）
- [ ] 无支付相关 API 调用（无 Stripe / 微信支付 / 支付宝）
- [ ] 隐私政策页 + 协议页**自身不挂**合规 banner（meta.hideBanner）

## §6 Onboarding 与首次体验（v14 新增）

**验收方式**：清空 localStorage 后首次访问

### 3 屏引导

- [ ] 首次访问首页（清 localStorage 后），自动弹出 onboarding 引导
- [ ] 引导有 3 屏：① 假外卖，真段子 ② 随便点，没成本 ③ 彩蛋藏在这
- [ ] 屏间可滑动切换（左右滑 + 下一步按钮 + 跳过按钮）
- [ ] 末屏「开始玩 🎭」按钮点击后关闭引导 + 写 `chaos_onboarded = 1`

### 第二次访问

- [ ] 第二次访问首页，**不再弹** onboarding（localStorage 已标记）
- [ ] 清 localStorage 后再次弹 onboarding（重新引导）

### 引导体验

- [ ] 整个引导 < 30 秒（3 屏 × ≤10 秒）
- [ ] 引导动画有"哇"的瞬间（emoji 浮动 bobble 动画）

## §7 评价闭环 v15（庆祝态 + 分享 + 催评 + AI 引导）

**验收方式**：完成 1 单 + 写评价，看完整交互

### 评价弹窗 AI 引导

- [ ] 打开评价弹窗，顶部有「✨ AI 已为你写好初稿，点下面直接编辑」提示
- [ ] 戏精评论文本自动预填（可编辑）
- [ ] 老板回复文本自动预填

### 提交后庆祝态

- [ ] 点「🎭 发布这出戏」后，切到庆祝态（不是直接关闭）
- [ ] 庆祝态显示「🎉 好戏已发布」+ 已发布的评价（评分+标签+文本+老板回复）
- [ ] 庆祝态有 pop 动画（cubic-bezier 弹跳曲线）
- [ ] 庆祝态有「📤 分享到朋友圈」按钮

### 分享按钮（3 档降级）

- [ ] 点击分享：优先调用 `navigator.share`（移动 Web）
- [ ] `navigator.share` 不可用时降级 `navigator.clipboard.writeText`
- [ ] clipboard 不可用时降级 `document.execCommand('copy')`
- [ ] 分享文本格式：「【胡闹外卖】给 XXX 打了个 X 星：「...」老板回复：... #胡闹外卖」
- [ ] 复制成功后有 toast「已复制戏精评价，去朋友圈贴一下 ✨」

### 催评徽标

- [ ] OrderHistory 中，状态 `completed` 且 `!review` 的订单显示「🎭 去评价」**红色脉冲徽标**
- [ ] 徽标有 pulse 动画（1.5s ease-in-out infinite）

## §8 数据埋点 v17（7 个事件 · 决策 #023）

**验收方式**：跑一遍流程后，DevTools → Application → LocalStorage → `chaos_metrics`

### 埋点写入正确性

- [ ] `onboarding_complete` 在 onboarding 末屏「开始玩」点击后写入
- [ ] `shop_click` 在 ShopCard 点击时写入（带 shopId + personality 附加数据）
- [ ] `dish_add` 在菜品「+」按钮点击时写入（带 dishId + category + price）
- [ ] `order_submit` 在 Checkout 提交订单时写入（带 shopId + itemCount + totalPrice + hasRemark + address）
- [ ] `review_submit` 在评价提交时写入（带 rating + tagCount）
- [ ] `review_share` 在分享按钮点击时写入（带 method：native / clipboard）
- [ ] `page_error` 在 JS 运行时错误 + Promise 未处理拒绝时写入

### 埋点稳定性

- [ ] 跑一遍完整流程后，`chaos_metrics` 累加 ≥ 5 个事件
- [ ] DevTools Console 跑 `dumpMetrics()` 能看到人话版输出

## §9 M3 上线后综合验收

**验收方式**：跨浏览器 + 真机测试 + 性能 + 异常路径

### 跨浏览器

- [ ] iOS Safari 14+：onboarding + 加购 + 下单 + 评价 + 分享降级到 clipboard 全流程通过
- [ ] Android Chrome 90+：同上
- [ ] 微信内置浏览器：分享按钮走 clipboard 降级（不支持 navigator.share）

### 真机测试

- [ ] 真机 iPhone（任一型号）：可正常操作，无明显卡顿
- [ ] 真机 Android（任一型号）：同上

### localStorage 健康检查

- [ ] localStorage 不可用时（隐私模式 / 满），onboarding 保守弹一次
- [ ] localStorage 写入失败时，metrics 不崩溃（try/catch 兜底）
- [ ] 多 tab 打开不会互相覆盖（localStorage 同步延迟可接受）

### 性能与稳定性

- [ ] DevTools Console **0 个红色 Error**
- [ ] 资源 Warning ≤ 3 个
- [ ] 首屏 < 3 秒（4G 模拟）
- [ ] 路由切换 < 1 秒

### 异常路径

- [ ] 购物车为空时点「去结算」：有提示（不直接跳转空的下单页）
- [ ] 页面刷新后，进行中的订单：重新进入订单页能看到最后状态
- [ ] localStorage 满时，metrics 自动停止写入（不抛错）
- [ ] 评价标签为空（未选）：提交按钮可点击，正常提交

### 品牌 & 文案

- [ ] 「零卡路里的外卖，越点越瘦」首页可见
- [ ] 顶部合规 banner 可见（半透明 + 胡闹橙主题）
- [ ] 没有遗留「TODO」「测试文字」「placeholder」等开发临时文案
- [ ] 4 处品牌元素全部到位：① Logo ② 首页 Slogan ③ 评价戏精标签 ④ 送达彩蛋页
