## 🤖 AI 后台维护方案（7-21 錡哥 · 暂不实现，先记录）

> **最终架构**：独立为 `whoknow-brain` 项目（胡闹控制中心），所有产品共用。
> 详见 `/home/node/clawd_code/projects/whoknow-brain/PROJECT-STATUS.md`

核心思路：AI 不上前台与玩家交互，只在后台跑每日维护，¥0.X/天。

### 场景 1：NPC 吐槽自动进化 🏆

```
每日分析「被点最多的菜」「用户备注高频词」「回头客数据」
  → AI 生成全新老板吐槽备选池
  → 用户第2次点单时收到全新对话
```

**例子**：
- 用户第1次点「胡闹拌面」：老板说"又是你？上周的还没吃完吧？"
- 用户第7次点同一家：老板说"我看看…你点这家第7次了。要不我入股你胃吧。"

### 场景 2：戏精骑手配送剧

```
根据订单内容 + 天气 + 时段
  → AI 生成骑手配送途中的「剧情段子」
  → 用户刷新订单状态时随机触发
```

**例子**：
- 订单状态「取餐中」→ 骑手备注："店里猫爬我身上了…请给点时间"
- 订单状态「配送中」→ 骑手备注："导航说前方5米有奶茶店…嗯我先绕一下"

### 场景 3：紧跟时事热点 🔥 錡哥拍板

```
AI 每天爬热搜 / 接受输入 →
  → NPC老板 + 骑手 + 菜品名 同步追热点
  → 用户下单时看到的是今天的段子
```

**例子**：
- 某明星塌房 → NPC老板："我给XXX写过应援代码…现在烧了"
- 高温预警 → 骑手备注："柏油路烫jio，建议加小费"
- 春运 → 老板："我去抢票了，店先关了，上面不代买"
- AI 圈大事 → 导购/老板："这AI比我能杠"
