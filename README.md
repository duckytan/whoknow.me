### AI 角色定位（最重要的一条）

> **AI 不上台当演员。AI 是导演。**

- 前台（用户看到的）→ NPC 讲段子、选招制、下单动效
- 后台（AI 做的）→ 写新段子、调平衡、追热搜、分析数据
- **所有 AI 调用集中在 whoknow-brain，一次预算养活全场**

### 传播设计

每个交互必须回答：**用户会截图这张页面发朋友圈吗？**

```
🚀 截图价值极高 → 发小红书/微博
✅ 值得截 → 发家族群/朋友
🟡 一般 → 砍掉这个交互
```

## 🧠 设计原则

### 用户思维铁律（7-20 錡哥金句 🔴 P0）

> **「你要学会从用户的思维去想问题，产品都是给用户使用的，你只有站在用户的视觉去看问题、思考问题，才能做出好产品。」**

任何产品决策前必须回答 3 个问题：
1. **用户能用吗？** — 30 秒内能学会吗？
2. **用户会爽吗？** — 用完会有"哇"的瞬间吗？
3. **用户会传播吗？** — 会想截图/发朋友圈/告诉朋友吗？

### 开发者定位（7-20 錡哥确认）

- **主体类型**：个人开发者，无营业执照
- **开发节奏**：没有固定工期，想到就做（所有方案自动避开需企业资质的方案）
