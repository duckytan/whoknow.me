# 🎨 whoknow · 品牌视觉规范 (BRAND.md)

> **胡闹宇宙的统一视觉语言地基。** 所有 `whoknow-*` 产品共享本规范。
> 最后更新：2026-07-24 · UI Designer
> 配套文件：`styles/design-tokens.css`（开发者可直接 `@import` 的令牌）

---

## 0. 本规范解决什么问题

whoknow 是多产品品牌系列，但视觉语言正在**分裂**：

| 界面 | 当前视觉 | 风险 |
|------|----------|------|
| 主站 `index.html` | 暗色霓虹风（黑紫底 + 绿/橙） | 自成一体 |
| 胡闹外卖 `whoknow-waimai` | 8 分相似美团（白橙） | 与主站割裂 |
| 胡闹导购 `whoknow-mart` | 概念：仿淘宝 | 暂无统一约束 |

**解决方案 = 双主题 + 品牌锚色**：

```
         ┌─────────────────────────────────────────┐
         │   品牌锚色（贯穿明暗，不可替换）          │
         │   胡闹绿 #6eda78 · 橙红 #ff7849 · 紫 #8b5cf6 │
         └───────────────┬───────────────┬──────────┘
                         │               │
            ┌────────────▼───┐   ┌───────▼──────────┐
            │ 宇宙暗色主题    │   │ 产品浅色主题      │
            │ （主站/品牌门面）│   │ （外卖/导购 App） │
            │ 霓虹·神秘·梗    │   │ 复刻+2分品牌点缀  │
            └────────────────┘   └──────────────────┘
```

- **宇宙暗色**用于 `whoknow.me` 入口、关于页——凸显"宇宙/神秘/梗"调性。
- **产品浅色**用于子 App——尊重用户习惯（美团白橙 / 淘宝橙红），但用品牌锚色做"胡闹徽标、戏精弹层、品牌按钮"实现"一眼认出是 whoknow"。
- 无论明暗，**品牌锚色 + 胡闹字体（Bungee / ZCOOL）+ 戏精文案**必须出现，保证人格统一。

## 1. 品牌人格与设计哲学

### 品牌三义合一
```
胡闹（中文动作） + Who knows?（英文疑问） + 家人们谁懂啊（社媒梗） = 天然传播体质
```

### 设计三大铁律（与产品铁律对齐，应用到 UI）
1. **用户习惯优先**：每个界面先问"用户在这个场景习惯怎么操作？"
   - 刷外卖 → 列表 + 商家卡 + 购物车
   - 逛导购 → 商品详情 + 加购 + 结算
   - 看品牌站 → 英雄区 + 故事 + 入口
2. **截图价值优先**：每个高光交互必须回答"用户会截图发朋友圈吗？"
   - 为截图优化：金句高亮、状态机戏剧性（骑手翻车）、博弈胜利/翻车瞬间、徽章成就
3. **零负担优先**：不肝、不氪、不学，打开即玩。

### 四大痛点滤网（任何视觉决策先过）
无聊 → 想笑 → 没钱也想消费 → 管不住手。界面若不能服务任一痛点，砍。

## 2. 色彩系统

### 2.1 品牌锚色（贯穿明暗，不可替换）

| 角色 | Token | HEX | 用途 |
|------|-------|-----|------|
| 胡闹绿 | `--brand-green` | `#6eda78` | 主品牌色、活力、梗点高亮、成功态点缀、Logo 点缀 |
| 橙红 | `--brand-orange` | `#ff7849` | 主行动色（CTA）、胡闹/警示、强调、品牌按钮 |
| 宇宙紫 | `--brand-purple` | `#8b5cf6` | 神秘/宇宙调、次级强调、渐变光晕 |

> 锚色在所有主题中保持一致。浅色产品里它们用于"点缀"，不取代功能色（如外卖的黄色主色）。

| `--bg-2` | `#120a1a` | 卡片/浮层背景 |
| `--bg-3` | `#1a1126` | 输入框/凹陷背景 |
| `--line` | `#1f1828` | 分割线 |
| `--line-2` | `#2a2236` | 强分割线/边框 |
| `--fg` | `#ededef` | 主文字 |

| Token | HEX | 用途 |
|-------|-----|------|
| `--bg` | `#f7f7f8` | 页面背景（浅灰，仿美团/淘宝） |

| `--fg-dim` | `#666666` | 次级文字 |
| `--fg-mute` | `#999999` | 弱文字/占位 |

> 浅色主题下，子产品仍可保留其"宿主"主色（外卖黄 `#FFD100`、导购橙红系），但**品牌锚色必须出现在导航栏 Logo、戏精弹层、品牌化按钮、成就徽章**上。

### 2.4 语义色（WCAG AA，明暗共用需提亮）

| 语义 | Token | HEX | 暗底文字色 |
|------|-------|-----|-----------|
| 成功 | `--c-success` | `#10b981` | `#6ee7b7` |
| 警告 | `--c-warning` | `#f59e0b` | `#fcd34d` |
| 错误 | `--c-error` | `#ef4444` | `#fca5a5` |
| 信息 | `--c-info` | `#3b82f6` | `#93c5fd` |

### 2.5 无障碍对比度（强制）
- 正文 ≥ **4.5:1**，大字（≥24px 或 ≥19px 粗）≥ **3:1**。
- 已核验：胡闹绿 `#6eda78` on `#0a0612` ≈ **9:1** ✓；橙红 `#ff7849` on 暗底配白字 ✓。
- 语义色在暗底需使用"暗底文字色"变体以满足 4.5:1。

## 3. 字体系统

### 3.1 字体阵容

| 角色 | 字体 | 用途 | 备注 |
|------|------|------|------|
| 展示 / Logo | **Bungee** | 品牌字、大标题、logo | 已用于主站，英文为主 |
| 中文标题 | **ZCOOL QingKe HuangYou**（站酷庆科黄油体） | 中文大字标题、海报感 | 圆润有梗 |
| 手写梗 | **ZCOOL KuaiLe**（站酷快乐体） | NPC 戏精金句、吐槽、彩蛋（正文级梗文本） | 圆润手写感、强可读、有"人味" |
| 笔刷 / 印章 | **Ma Shan Zheng**（马善政毛笔楷书） | 印章、签名、强戏精台词、笔刷装饰 | 毛笔楷书仍可读，比草书正式 |
| 正文 / UI | **Inter** + **Noto Sans SC** | 所有界面正文与 UI 文本 | 中英混排首选 |
| 等宽 / 标签 | **JetBrains Mono** | 标签、数据、代码感、mono 徽章 | 科技/梗感 |

> ⚠️ **退役字体**：原「柳建毛草 Liu Jian Mao Cao」为草书，连笔难辨，**正文 / 梗文本一律禁用**。手写梗改用站酷快乐体（可读 + 有梗），印章 / 笔刷用马善政毛笔楷书（毛笔楷书仍可读）。

```css
@import url('https://fonts.googleapis.com/css2?family=Inter:wght@300;400;600;800;900&family=Noto+Sans+SC:wght@400;700;900&family=JetBrains+Mono:wght@400;600&family=Bungee&family=ZCOOL+QingKe+HuangYou&family=ZCOOL+KuaiLe&family=Ma+Shan+Zheng&display=swap');
```

## 6. 阴影与层级

**暗色主题**：用"低光晕"替代投影，体现霓虹感。
- `--shadow-glow-green`：`0 0 24px rgba(110,218,120,.25)`
- `--shadow-glow-orange`：`0 0 24px rgba(255,120,73,.25)`
- `--shadow-card`：`0 8px 24px rgba(0,0,0,.4)`

**浅色主题**：标准投影。
- `--shadow-sm`：`0 1px 2px rgba(0,0,0,.05)`
- `--shadow-md`：`0 4px 12px rgba(0,0,0,.08)`
- `--shadow-lg`：`0 12px 32px rgba(0,0,0,.12)`

## 7. 组件库基础

> 所有组件遵循：默认态 → hover → active(press) → focus-visible → disabled。交互态必须有视觉反馈。

### 7.1 按钮 Button
| 变体 | 外观 | 用途 |
|------|------|------|
| 主按钮 `--btn-primary` | 橙红填充 `var(--brand-orange)` / 白字 | 主 CTA（下单、结算） |
| 次按钮 `--btn-secondary` | 描边 / 浅底 | 次级动作 |
| 幽灵按钮 `--btn-ghost` | 透明 + 文字色 | 低强调 |
| 品牌绿按钮 `--btn-green` | 绿填充 | 成功/品牌强调（如"已胡闹"） |
| 危险按钮 `--btn-danger` | 红填充 | 删除/放弃 |

- 尺寸：sm(32h) / md(40h) / lg(48h)；圆角 `--radius-md`；最小触控 44×44。
- hover：上移 1px + 阴影；active：下移归位 + 压暗；focus-visible：`outline: 2px solid var(--brand-green); outline-offset:2px`。
- **胡闹态**：下单成功可触发按钮"抖动 + 绿光晕"微动效。

### 7.2 输入框 Input
- 浅色：`bg-3` 底 + `--line-2` 边 + `--radius-md`；focus：边框转 `--brand-orange` + `box-shadow 0 0 0 3px rgba(255,120,73,.15)`。
- 暗色：focus 边框转 `--brand-green` + 绿光晕。
- 必带 `<label>` 与占位提示；错误态：红边 + 下方红字说明（非仅靠颜色）。

### 7.3 卡片 Card
- 商家卡 / 商品卡 / 段子卡：白（浅色）或 `bg-2`（暗色）底，`--radius-lg`，轻阴影。
- hover：上移 2px + 阴影加深（浅色）/ 绿光晕（暗色）。
- 段子卡：用 `ZCOOL KuaiLe` / `Ma Shan Zheng` 手写体承载 NPC 金句，配"可截图"高亮背景（**禁用柳建毛草等草书字体——不可读**）。

### 7.4 徽章 / 标签 Badge & Tag
- mono 徽章：`JetBrains Mono` + `--radius-pill` + 描边，用于"今日梗 / HOT / 已翻车"。
- 戏精标签：圆角药丸 + 品牌色文字，承载吐槽分类（毒舌/理性/散漫…）。

### 7.5 导航 Navigation
- 产品 App：**底部 Tab Bar**（仿美团/淘宝），图标 + 文字，选中态用品牌橙红。
- 主站：顶部极简导航 + 锚点；移动端 `max-width:640px` 居中，手机感。

### 7.6 反馈 Feedback
- 戏精弹层 Dialog：弹簧入场，NPC 头像 + 手写体台词 + 选招/确认按钮（用于导购博弈、老板回复）。
- Toast：底部轻提示，2s 自动消失，承载"下单成功""已胡闹"。
- 骨架屏 Skeleton：浅灰脉冲占位，加载商家/商品列表。

## 8. 动效与微交互

- **原则**：动效服务于"戏"与"反馈"，不炫技；单动作 ≤ 300ms。
- 微交互：hover 上移 1–2px + 阴影；按钮 press 缩放 `.97`。
- 戏精动效：弹层弹簧（`cubic-bezier(.34,1.56,.64,1)`）、台词逐字浮现、骑手 `rider_lost` 抖动。
- **无障碍**：所有动效包在 `@media (prefers-reduced-motion: reduce)` 中降级为瞬时。

## 9. 响应式断点

移动优先。`whoknow.me` 主站 `max-width:640px` 居中（手机优先）；子 App 全宽移动布局。

| 断点 | 范围 | 策略 |
|------|------|------|
| 移动 | 320–639 | 基准，单列 |
| 小屏 | 640+ | 主站容器 640；App 仍单列 |
| 中屏 | 768+ | 列表两列 |
| 桌面 | 1024+ | 多列网格、留白增大 |
| 大屏 | 1280+ | `max-width:1280` 居中 |

## 12. Logo 与品牌标识规范

> Logo 是"一眼认出是 whoknow"的最后防线，必须严格受控。

**12.1 构成**
- 主标识：`Bungee` 全小写字标 **whoknow**（允许字距 `-1%` 收紧）。
- 宇宙符号（可选）：字标后可接「`?`」或星点，仅用于品牌门面，**不进产品 App 标题栏**。
- 中文副标（可选）：`ZCOOL QingKe HuangYou` 的"胡闹宇宙"，仅在大屏 Hero 与关于页出现。

**12.2 安全间距（Clear Space）**
- 四周留白 ≥ 字标高度的 **1/4**（记为 `h/4`）。任何元素不得侵入此区。

**12.3 最小尺寸**
- 数字界面：字标高度 ≥ **24px**（低于则改用锚色圆形「?」标记）。
- 印刷物料：高度 ≥ **12mm**。

**12.4 合法变体**
| 变体 | 适用 | 说明 |
|------|------|------|
| 锚色彩标 | 浅色背景 | `whoknow` 用 `--brand-orange`，`?` 用 `--brand-green` |
| 单色标（fg） | 任意背景 | 使用 `--fg`，保持可读 |
| 反白标 | 暗底/彩图 | 使用 `#ffffff` 或 `--bg` 反相 |

**12.5 与宿主 App 共存（关键）**
- 子产品图标 = **宿主形状**（如外卖圆角方块）+ **whoknow 角标**：右下 `16px` 圆，绿底 `--brand-green` + 白「?」。
- 标题栏左：宿主名（如"美团"风格）+ 角标；**不得**用 Bungee 整段替换宿主 logo，以免侵权与违和。

**12.6 错误用法（禁止）**
拉伸 / 倾斜 / 改色（除 §12.4 三变体外）/ 加描边 / 加投影 / 旋转 / 改字距超过 ±2% / 在杂乱背景上去安全区。

## 13. 图标系统 Iconography

**13.1 网格与构造**
- 画布 **24×24**，内容区留白 **2px**，关键线宽 **2px**（浅色）/ **1.5px**（暗色，配锚色 glow）。
- 端点 `round`，拐角圆化 `2px`；视觉居中修正（光学平衡，非几何中心）。
- 导出倍率 `@1x/@2x/@3x`，禁用位图缩放。

**13.2 两类图标**
| 类型 | 风格 | 颜色 | 用途 |
|------|------|------|------|
| 功能图标 | 线性 1.5–2px | `--fg-dim`（默认）/ 选中 `--brand-orange` | 导航、操作、状态 |
| 戏精图标 | 实心圆底 + 锚色 + emoji/符号 | `--brand-green` 或 `--brand-orange` 底 + 白符 | NPC、吐槽分类、成就 |

**13.3 尺寸与触控**
- 显示尺寸：`16 / 20 / 24px`；图标按钮触控区 **≥ 44×44px**（用透明 padding 撑）。
- 禁用：不对齐网格、线宽不统一、在彩图上用浅灰不可读。

## 15. 栅格与布局系统 Grid & Layout

**15.1 基准**
- 基准 **4px**；列栅格 **8pt**（列宽 = 8 的倍数）。
- 主站：单列，`--container-max: 640px` 居中，手机优先，左右 padding `--sp-4`(16)。
- 子 App：全宽移动布局，顶部/底部安全区用 `env(safe-area-inset-*)`，内容 max `480px` 居中（大屏不拉巨）。

**15.2 桌面拓展（品牌站/后台）**
- 12 列网格，列间距 `--sp-6`(24)，外边距 `--sp-8`(32)，页面 max `1280px`。
- 区块垂直节奏：`--sp-12`(48) 起，Hero 用 `--sp-16`(64)。

**15.3 Z 轴层级（统一变量）**
| Token | 值 | 用途 |
|-------|----|------|
| `--z-base` | 0 | 内容 |
| `--z-sticky` | 100 | 顶/底导航、吸顶 |

| `--z-modal` | 2000 | 戏精弹层、Dialog |
| `--z-toast` | 3000 | Toast、临时提示 |

## 16. 动效参数明细表 Motion Spec

> 动效服务于"戏"与"反馈"，单动作 ≤ 300ms；全部包 `@media (prefers-reduced-motion: reduce)` 降级瞬时。

| 交互类型 | Duration | Easing | 位移 / 透明度 |
|----------|----------|--------|--------------|
| hover 反馈 | 150ms | `--ease-smooth` | `translateY(-1~2px)` + 阴影加深 |
| 按钮 press | 100ms | `--ease-smooth` | `scale(.97)` |
| 卡片 hover | 200ms | `--ease-smooth` | `translateY(-2px)` + 光晕 |
| 戏精弹层入 | 300ms | `--ease-spring` | `opacity 0→1` + `translateY(8px→0)` + `scale(.96→1)` |
| 抽屉/抽屉出 | 250ms | `--ease-smooth` | `translateX(100%→0)` |
| Toast | 200ms | `--ease-smooth` | `translateY(100%→0)` → 2s 后反向消失 |
| 骨架屏 | 1.2s 循环 | — | `opacity .5↔1` 脉冲 |
| 页面/路由转场 | 250ms | `--ease-smooth` | `opacity` 淡 + 轻微 `translateY(8px)` |

## 17. 完整无障碍清单（WCAG AA 基线）

UI Designer 交付硬性要求，逐条 QA：

- [ ] **对比度**：正文 ≥ **4.5:1**，大字（≥24px / ≥19px 粗）≥ **3:1**（见 §2.5 已核验组合）。
- [ ] **键盘可达**：所有交互可纯键盘完成，无"卡死"焦点。
- [ ] **焦点顺序**：DOM 顺序 = 视觉顺序；`focus-visible` 用 `--focus-ring`（绿/橙 2px offset 2px）。
- [ ] **屏幕阅读器**：语义标签（`button`/`nav`/`main`）+ 必要 `aria-label`；装饰图标 `aria-hidden`。
- [ ] **触控目标**：所有可点元素 **≥ 44×44px**（§13.3）。
- [ ] **文字缩放**：支持浏览器 **200%** 缩放不破版（用 `rem`/相对单位，禁固定 `px` 高）。
- [ ] **错误预防**：表单有 `<label>` + 实时校验 + 错误文案（非仅靠颜色，§7.2）。
- [ ] **动效降级**：`prefers-reduced-motion` 下全部瞬时（§16 / tokens §5）。
- [ ] **颜色不独载**：状态变化必带形状/文字/图标辅助（不只换色）。
- [ ] **语言声明**：`<html lang="zh-CN">`。
- [ ] **锚色可读**：锚色仅作点缀/大色块，不承载小号正文。
