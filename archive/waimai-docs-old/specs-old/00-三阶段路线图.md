# 🗺️ 胡闹外卖 - 三阶段路线图

> **最后更新**：2026-07-20
> **决策人**：锡哥
> **核心原则**：每阶段端到端可用，逐步上线验证

## 🎯 整体战略

> **"每一步都能独立验证，不要憋大招"**

```
Phase 1 (12 天)  →  验证情绪产品力
Phase 2 (+2 周)  →  验证双边经济可行性
Phase 3 (+4-6 周) → 完全三边真人的"胡闹厨房"
```

---

## 📅 Phase 1：MVP 验证期（Week 1-2）

### 🎯 目标
**验证"披着美团外衣的虚拟外卖"这个产品形态有没有人买单**

### 👥 用户构成
- 买家 = 真人 ✅
- 商家 = NPC（20 个写死）
- 骑手 = NPC（8 个写死）

### ✅ 成功指标（上线 1 周后）
- [ ] DAU ≥ 100
- [ ] 平均会话时长 ≥ 5 分钟
- [ ] 单用户下单次数 ≥ 3
- [ ] 戏精评价率 ≥ 30%
- [ ] 分享/截图率 ≥ 5%

### 🛠 技术栈
```
Vue 3 + Vite + Vant UI 4 + Pinia + SCSS
数据：JSON 写死 + localStorage
部署：Vercel 免费
```

### 📋 核心功能
- [ ] 美团 UI 1:1（首页/商家列表/详情/购物车/订单/个人中心）
- [ ] 三 Tab 入口（买家 + 商家 NPC 演示 + 骑手 NPC 演示）
- [ ] 真实下单流程
- [ ] NPC 商家/骑手状态机 + 戏精甩话
- [ ] 骑手地图移动 SVG 动画
- [ ] 戏精评价系统
- [ ] 虚拟币（签到 + 订单奖励 + 看广告）

### 🚨 风险
- 数据不够丰富 → 准备 30 商家备用
- Vercel 慢 → Netlify 备选
- 戏精评价套路 → 收集真实评论当素材

---

## 📅 Phase 2：骑手真人化（Week 3-4）

### 🎯 目标
**验证"真人骑手 + NPC 商家"的可行性，验证双边经济系统**

### 👥 用户构成
- 买家 = 真人 ✅
- 商家 = NPC（保持）
- 骑手 = **真人**（新开放）

### ✅ 成功指标（上线 2 周后）
- [ ] 同时在线骑手 ≥ 20 人
- [ ] 真人骑手抢单率 ≥ 60%（NPC 兜底 ≤ 40%）
- [ ] 骑手平均单收益 ≥ X 虚拟币
- [ ] 骑手次日留存 ≥ 30%
- [ ] 买家投诉率 ≤ 5%

### 🛠 技术增量
```
后端：Node.js + Express + SQLite
实时：WebSocket（骑手抢单实时同步）
认证：手机号/邮箱登录
```

### 📋 新增功能
- [ ] 骑手=真人独立页面/客户端
- [ ] 抢单大厅（WebSocket 实时刷单）
- [ ] 骑手收益提现（虚拟币转账）
- [ ] NPC 骑手智能兜底（按在线真人数量自动切换）
- [ ] 骑手信誉分（影响接单优先级）

### 🚨 风险
- 没人当骑手 → NPC 兜底 100% 接管
- 骑手刷单 → 设备/IP 检测
- 实时性卡顿 → 降级到 5 秒轮询

---

## 📅 Phase 3：完全三边真人（Month 2-3）

### 🎯 目标
**真正的"胡闹厨房"——三玩家分工扮演**

### 👥 用户构成
- 买家 = 真人 ✅
- 商家 = **真人**（新开放）
- 骑手 = 真人 ✅
- NPC = 三方兜底（动态比例）

### ✅ 成功指标（上线 1 个月后）
- [ ] 同时在线玩家 ≥ 500
- [ ] 三边真人成交率 ≥ 70%
- [ ] 日订单量 ≥ 1000
- [ ] 月留存 ≥ 20%
- [ ] 商家/骑手/买家活跃比例 3:2:5（健康比例）

### 🛠 技术增量
```
撮合引擎：订单分配算法（按距离/信誉/历史）
信誉系统：三套互通信誉体系
商家后台：独立 Web 端（装修/菜单/定价）
实时状态广播：三方位同步
投诉/仲裁：玩家陪审团机制
```

### 📋 新增功能
- [ ] 商家开店系统（装修/菜单编辑/定价/库存）
- [ ] 商家资质审核（虚拟）
- [ ] 平台抽成系统
- [ ] 三方互评（卖家给骑手、骑手给买家、买家给商家）
- [ ] 信用分/段位系统
- [ ] 投诉仲裁机制
- [ ] 工会/车队系统
- [ ] 排行榜/赛季

### 🚨 风险
- 三方不可能同时有玩家 → NPC 按比例兜底，**永远不让玩家卡单**
- 经济通胀 → 抽成 + 税收 + 维护费回收
- 恶意刷单 → 设备/IP/信誉综合检测
- 商家菜品违规 → 关键词过滤 + 玩家举报

---

## 🎯 Phase 间的"止损线"

| 阶段 | 止损线 | 怎么办 |
|------|--------|--------|
| **Phase 1 失败** | DAU < 50 / 评价率 < 10% | 调整定位，可能换赛道 |
| **Phase 2 失败** | 真人骑手 < 5 在线 | 永久骑手 NPC，砍掉 Phase 2 |
| **Phase 3 失败** | 三边比例 < 1:1:2 | 保留 Phase 2 形态，扩大运营 |

---

## 💡 每阶段的"价值底线"

- **Phase 1 的成功标准**：让人笑/上瘾/截图分享 → 情绪价值
- **Phase 2 的成功标准**：让骑手赚到钱 → 经济价值
- **Phase 3 的成功标准**：让三边玩家互相依赖 → 社交价值
