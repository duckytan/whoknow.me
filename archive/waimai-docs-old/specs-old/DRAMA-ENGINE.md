## 一、现状分析

### 当前 NPC 系统

```
triggerOrderFlow(orderId)
  ├─ 固定 5 个骑手，随机分配
  ├─ 固定 25% 概率老板发疯
  ├─ 20% 概率骑手迷路 (r003 固定迷路)
  ├─ 台词只依赖：shopName / dishName / riderName / personality
  └─ 完全随机，不受用户行为影响
```

**现有数据资产**：
- 15 家店铺（5 种性格）
- 92 道菜品
- 160+ 评价模板 + 26 条老板回复
- 5 类骑手 + 送达模板
- 5 个胡闹彩蛋

**当前局限**：
1. 所有订单剧情一模一样（只是台词随机）
2. 用户的操作（地址/备注/历史）零影响
3. 骑手挑选 = 纯随机
4. 没有"剧情记忆"——同一家店吃 10 次，老板永远一副初次见面的态度

## 二、目标定义

> **每个订单都是一场独一无二的「微型戏剧」**

用户的所有行为 → 输入剧情引擎 → 输出个性化 NPC 演出

### 输入变量池

| 类别 | 变量 | 来源 | 例子 |
|------|------|------|------|
| **下单信息** | `address` | 下拉框选择 | 家庭/公厕/ICU |
| **下单信息** | `remark` | 下拉框选择 | "多放辣"/"别骂了" |
| **下单信息** | `dishCount` | 购物车 | 点了 1 个菜 vs 8 个菜 |
| **下单信息** | `totalPrice` | 购物车 | ¥28 vs ¥328 |
| **下单信息** | `timeOfDay` | 自动 | 凌晨/午市/深夜 |
| **历史信息** | `shopOrderCount` | localStorage | 第 1 次光顾 vs 第 10 次 |
| **历史信息** | `dishRepeatCount` | localStorage | 某菜回购次数 |
| **历史信息** | `lastRemark` | 上次订单备注 | 上次备注"多放辣" |
| **历史信息** | `totalOrders` | localStorage | 用户总点单数 |
| **场景信息** | `bossPersonality` | 店铺数据 | angry/gentle/weird/lazy/philosophical |

## 三、引擎架构

```
┌─────────────────────────────────────────┐
│           订单提交（Checkout）            │
│  地址 · 备注 · 购物车 · 时间 · 历史       │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│        剧情上下文构建器                   │
│  buildDramaContext(orderInput, history)  │
│  ─────────────────────────────────────   │
│  输出: DramaContext                      │
│  {                                      │
│    address: '百慕大三角洲',               │
│    remark: '不要香菜, 多放辣',             │
│    dishCount: 3,                         │
│    totalPrice: 128,                      │
│    timeSlot: 'dinner',                   │
│    isNewcomer: false,                    │
│    shopVisitCount: 5,                    │
│    dishRepeatMap: { d001: 3, d003: 5 }, │
│    lastRemark: '多放辣',                  │
│    totalOrderCount: 23,                  │
│  }                                       │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│          NPC 状态机（增强版）              │
│  triggerOrderFlow(orderId, context)      │
│  ─────────────────────────────────────   │
│  1. 变量影响 → 概率调整表                 │
│  2. 概率 → 事件选择                      │
│  3. 事件 + 上下文 → 台词模板匹配          │
│  4. 台词 + 变量注入 → NPC 演出            │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│    订单时间线（与现在一致）                │
│  接单 → (吐槽) → 出餐 → 抢单 → 配送 → 送达 │
│  ↓        ↓        ↓     ↓      ↓    ↓   │
│  老板    老板骂   厨房   骑手   迷路   彩蛋 │
└─────────────────────────────────────────┘
```

## 四、变量影响矩阵

### 4.1 地址影响

| 地址 | 触发标签 | 效果 |
|:----:|:--------:|------|
| 🏠 **家庭** | `home` | 配送时间 -10%，骑手台词"送回家啊，温馨" |
| 🏫 **学校** | `school` | 老板："学生仔？给你多加点" |
| 🏢 **公司** | `company` | 配送时间 +15%（门禁等），老板："打工人啊，辛苦了" |
| 🚽 **公厕** | `toilet` | 老板吐槽概率 +30%，"送到公厕？？" |
| 🔺 **百慕大三角洲** | `bermuda` | 骑手迷路概率 +60%，配送时间 +30% |
| 🏥 **ICU 病房** | `icu` | 骑手台词温和，老板："吃完早点康复" |

### 4.2 备注影响

| 备注类型 | 触发标签 | 效果 |
|:--------:|:--------:|------|
| 正常 | `normal` | 无特殊影响 |
| 多放辣 | `more_spicy` | 老板暴躁概率 +15%，"辣死你我可不管" |
| 少放辣 | `less_spicy` | 老板："吃不了辣你来我店？？" |
| 不要香菜 | `no_coriander` | 老板骂骂咧咧摘香菜 |
| 别骂了 | `no_scold` | 老板暂停骂人 30s，然后变本加厉 |
| 表演个才艺 | `show_time` | 骑手配送中途台词变"才艺展示"风格 |

### 4.3 历史影响

| 条件 | 触发标签 | 效果 |
|:----:|:--------:|------|
| 首次光顾 | `new_customer` | 老板："新面孔啊，给你做认真点" |
| 回头客（≥3 次）| `regular` | 老板："又是你？" 概率 +40% |
| 老顾客（≥10 次）| `vip` | 专属 VIP 台词 |
| 某菜买 ≥3 次 | `dish_regular` | "这道菜你点了 X 次了，我闭眼都会做" |
| 某菜买 ≥10 次 | `dish_vip` | "你对这个菜是真爱" |
| 累计下单 ≥10 | `old_friend` | 骑手："老顾客了，优先配送" |
| 上次差评 | `last_bad_review` | 老板："上次给一星的是不是你？" |

### 4.4 环境影响

| 条件 | 触发标签 | 效果 |
|:----:|:--------:|------|
| 凌晨 0-6 点 | `late_night` | 骑手："半夜点外卖的人都有故事" |
| 午餐 11-13 点 | `lunch_rush` | 配送时间 +10%，老板："忙死了催什么" |
| 晚餐 17-19 点 | `dinner_rush` | 同午餐 |
| 周末 | `weekend` | 老板："周末也不让我休息" |

### 4.5 数量影响

| 条件 | 触发标签 | 效果 |
|:----:|:--------:|------|
| 1 个菜 | `single_dish` | "就点一个菜？看不起谁" |
| ≥5 个菜 | `feast` | 老板发疯概率 +20%，"点这么多？你养猪呢" |
| ≥8 个菜 | `banquet` | 彩蛋触发概率 +30% |

## 五、概率计算引擎

```
最终概率 = 基础概率 + Σ(各变量偏移)

示例：骑手迷路概率
  基础概率: 20%
  + 地址=百慕大三角洲: +60%
  + 骑手=张迷路(r003): +40%
  + 配送时间=午餐高峰: +10%
  ─────────────────────
  最终 = 20% + 60% + (如果选中 r003 则+40%) + (如果午餐则+10%)
```

**核心公式**（所有事件通用）：

```
P(event) = clamp(baseP + Σ(modifiers), 0, 100)
```

## 六、台词模板匹配

在现有模板系统上增加 **条件标签**：

```json
{
  "boss": {
    "angry": [
      { "text": "催什么催！锅还没热呢！", "tags": ["any"] },
      { "text": "又是你？你是来蹭骂的吧？", "tags": ["regular"] },
      { "text": "新面孔？先挨顿骂再吃", "tags": ["new_customer"] },
      { "text": "送到公厕？？我做的菜不配去公厕", "tags": ["toilet"] },
      { "text": "ICU 的？给你做清淡点", "tags": ["icu"] },
      { "text": "凌晨点外卖，你不睡觉我不用睡？", "tags": ["late_night"] }
    ]
  }
}
```

**匹配逻辑**：
1. 当前 context 命中哪些 tags
2. 筛选出匹配的台词
3. 按优先级排序（精确匹配 > 通配 any）
4. 随机选一条

## 十一、示例：一个完整的订单剧情

```
用户选择了：
  · 地址：百慕大三角洲
  · 备注：多放辣
  · 商品：老王烧烤 ×5 个菜
  · 历史：第 3 次来老王烧烤

剧情引擎计算：
  · 骑手：随机（百慕大自带迷路 tag）→ r003 张迷路
  · 老板：暴躁 + 多放辣 + 老顾客
  · 迷路概率：基础 20% + 百慕大 60% + r003 40% = 120% → 100% 迷路

NPC 演出时间线：
  [2s]  老板：「又是你？第 3 次了吧，给你多加点辣，辣不死你」
  [10s] 老板发疯中：「点这么多菜？你喂猪呢？！」
  [11s] 系统：「虽然老板在发疯，但菜还是做好了」

  [22s] 骑手：「我不小心骑进了百慕大，导航失灵了！」
  [30s] 骑手送达：「我也不知道我怎么到的，缘分吧」
  [彩蛋] 百慕大特殊彩蛋：「你的外卖穿越了时空，比预计早了 100 年到达」
```
