## 🔧 怎么埋点（个人开发者最简方案）

### 方案 A · localStorage 自埋点（**最推荐 · 零成本**）

**机制**：所有事件累加到 `localStorage.chaos_metrics` 一个 JSON 对象里。

**优点**：
- ✅ 零成本（无后端、无第三方）
- ✅ 简单（30 行代码搞定）
- ✅ 调试方便（DevTools 直接看）

**缺点**：
- ❌ 数据只在用户本地（你看不到）
- ❌ 清浏览器 = 数据清零
- ❌ 不能跨设备看数据

**代码**（建议）：

```ts
// src/utils/metrics.ts
export function track(event: string, data?: Record<string, any>) {
  try {
    const key = 'chaos_metrics'
    const raw = localStorage.getItem(key) || '{}'
    const metrics = JSON.parse(raw)
    metrics[event] = (metrics[event] || 0) + 1
    if (data) metrics[`${event}_data`] = data
    localStorage.setItem(key, JSON.stringify(metrics))
  } catch {}
}

// 用法：
track('onboarding_complete')
track('shop_click', { shopId: 's001' })
track('order_submit', { shopId: 's001', itemCount: 3 })
```

**数据怎么看**：让你（锡哥）跑一遍 → DevTools → Application → LocalStorage → `chaos_metrics` → 截图发我。

### 方案 B · 简易后端统计（Supabase 免费层）

**机制**：埋点 POST 到 Supabase 数据库。

**优点**：
- ✅ 数据集中
- ✅ 跨设备看
- ✅ 5 分钟搭好

**缺点**：
- ❌ 要写后端（Supabase 配 JS SDK）
- ❌ 用户请求多了会超免费层（500MB）
- ❌ 个人开发者没 ICP = 国内 Supabase 慢

**结论**：MVP 阶段过度设计，等用户上 1000 再考虑。
