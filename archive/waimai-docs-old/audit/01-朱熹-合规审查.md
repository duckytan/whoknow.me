## 第 0 步 · Pre-Mortem（项目启动前必做）

### 假设失败时点
**6 个月后**，胡闹外卖已经"彻底凉了"。

### 想象失败情境
- 上线 2 周后 DAU < 30，戏精评价没人写
- 应用商店审核被拒，理由"仿冒美团/京东外卖商标"
- Vant UI 出现安全漏洞，被人挖出来刷 XSS
- Vercel 部署地区被墙，国内访问 5 秒+
- 像素风 + 美团 UI 视觉严重撕裂，用户吐槽"四不像"

### 倒推失败原因（10 条）
1. **市场验证失真**：FoodNeverComes 在韩国爆火 ≠ 中国能爆（独居率/文化差异）
2. **戏精评价过度套路**：50 条模板 1 周就被看穿，"哦又来了"
3. **应用商店审核没跑通**：UI 1:1 复刻美团 = 商标侵权
4. **Vant UI 第三方依赖**：Vant 4.x 实际维护频率下降，某次小版本破坏性更新
5. **数据写死**：20 NPC 商家，玩家 3 天刷完，无重玩价值
6. **单边真人冷启动**：Phase 1 只有买家，没人陪他玩 = 戏精评价发给谁看？
7. **本地化偏差**：把韩国"다巴胺"概念翻译成"胡闹"，情绪感染力折损
8. **Vercel 国内访问**：被墙或限速，首屏 5s+ = 80% 用户流失
9. **纯前端无后端**：localStorage 无法跨设备 = 用户第二天回来清零 = 留存=0
10. **像素风 vs 美团 1:1 矛盾**：决策 #002 写了"像素风"，但 #005 写了"美团 1:1"，决策互相打脸

### 高风险信号（🔴 critical 必查项）
- **应用商店审核**：UI 1:1 复刻美团 = 商标侵权风险
- **决策冲突**：#002 像素风 vs #005 美团 1:1 = 自我矛盾
- **单边玩家**：只有买家玩 = "自嗨游戏"
- **Vercel 国内访问**：技术依赖的合规与可用性

### 已预防措施
- ✅ 决策 #002 加补丁"Phase 1 先美团 1:1"
- ✅ 不依赖仿站项目（从零搭 Vue3）
- ⚠️ 应用商店合规未拍板
- ⚠️ Vercel 国内访问未测

## 第 2 步 · 否定性对抗（5 假设 + 5 追问）

### 假设 1：FoodNeverComes 70 万人 = 中国 100 万市场
**类型**：Correlation（共存 ≠ 因果）
- 韩国土壤特殊（独居率/多巴胺文化）+ 70 万是累计不是日活
- **追问**：如果搬到中国，DAU 是 100 万还是 1000？
- **否定证据**：8 家 AI 调研中只有 3 家（千问/Mavis/Kimi）把 FNC 当标杆，其他 5 家没提 → **共识不强**
- **判定**：⚠️ Conditional — 假设 70 万累计 ≈ 中国 1-10 万 DAU，**不是 100 万**

### 假设 2：UI 1:1 复刻美团 = 学习成本 0
**类型**：Hidden（隐藏前提）
- 隐藏前提：用户**能看出**这是美团 UI
- **追问**：如果 Vant UI 跟美团有 5% 差异（颜色/字体/间距），用户能一秒识别吗？
- **否定证据**：Vant UI 4 是**通用移动端**组件库，不是专门仿美团。要做到 1:1 = **自己改 Vant 主题色 + 自写组件** = 工作量翻倍
- **判定**：⚠️ Conditional — 是"美团风"不是"美团 1:1"，要诚实标定

### 假设 3：Vant UI = 维护 0 成本
**类型**：Authority（信了"有赞大厂开源"）
- 隐藏前提：有赞会一直维护 Vant
- **追问**：Vant 4 最后一次发版是 2024 年？维护频率如何？
- **判定**：❌ **未验证**——这是 7-16 ABRT 教训类盲点：**信第三方维护 ≠ 真维护**
- **判定**：⚠️ Conditional — 需要核验 Vant 4 最近 6 个月 commit 频率

### 假设 4：localStorage = Phase 1 够用
**类型**：Temporal（过去对，假设现在还对）
- 隐藏前提：用户每天在同设备打开
- **追问**：如果用户用手机下了一半 → 电脑继续？手机换机？
- **判定**：❌ Fails — localStorage **不能跨设备**，DAU ≥ 100 时会出现大量"昨天下的订单去哪了"

### 假设 5：单边真人（只买家玩）= 验证情绪产品力
**类型**：Hidden
- 隐藏前提：戏精评价是"自评自乐"
- **追问**：用户给谁看戏精评价？如果没人看 = **自嗨 = 留存必崩**
- **判定**：⚠️ Conditional — 需要"分享给朋友"路径，截图按钮必须显眼

## 第 5 步 · 推荐行动（必走 3 项）

### 🔴 P0 #1：应用商店合规自查
- **预期效果**：避免上架被拒 + 商标诉讼
- **行动**：
  - 用「胡闹外卖」「ChaosWaimai」查国家知识产权局商标局
  - 改 UI：**不能让用户一眼以为是美团**——顶部品牌区写明"胡闹外卖 · 游戏"
  - 应用商店分类选"游戏"（不是"美食"）
- **验证方法**：把 UI 截图发给 5 个不看新闻的朋友，问"这是什么 App？"——答案不该是"美团"
- **回退**：上 Web 端（PWA），绕开应用商店审核

### 🔴 P0 #2：解 #002/#005 矛盾
- **预期效果**：内部决策一致 = 代码方向不漂
- **行动**：
  - 决策 #002 改为：「Phase 1 美团 1:1 复刻（像素风留 Phase 3）」
  - 或者决策 #005 改为：「像素风美团化（Vant UI 不够 1:1）」
- **验证方法**：决策日志 grep"像素风"和"美团"看是否互斥
- **回退**：暂时两个都做（美团风 + 像素风装饰条），但要标 P2

### 🟡 P1 #3：localStorage 边界声明
- **预期效果**：Phase 1 用户知道"换设备=重新开始"是产品设计而非 bug
- **行动**：
  - 写 onboarding 文案："这是单设备游戏，换手机=重新开始"
  - 加"导出存档"功能（导出 localStorage 为 JSON）
- **验证**：第 7 天用户调研"你知道换手机会清空吗" ≥ 80% 知道
