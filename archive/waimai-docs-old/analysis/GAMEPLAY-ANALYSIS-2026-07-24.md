# 🎮 胡闹外卖 · 游戏玩法分析报告

> **分析人**：玩法师（Game Designer Agent）
> **日期**：2026-07-24
> **对象**：`whoknow-waimai/` 当前代码 + `PROJECT-STATUS.md` / `DEVELOPER.md` / `docs/` 验收与审计文档
> **方法**：源码通读（引擎层 + 状态层 + 页面层 + 数据层）+ 设计文档交叉验证。**非真机 playtest**——本文是设计层分析，第 0 条建议是补一次真机试玩。
> **一句话结论**：
> **作为「概念验证 / 一次性爽产品」——玩法极其合适，甚至超出预期。** 喜剧引擎、把「备注/地址」做成 NPC 反应输入、节奏调速、截图分享点，都精准命中了 Phase 1 目标（验证胡闹系列能不能吸人）。
> **但作为「有留存循环的游戏」（四大痛点与 M2–M4 路线图隐含的诉求）——当前玩法还不成熟。** 它是一台精良的「一次性玩具」，长期脊柱偏细：没有失败、没有技巧、没有真正的社会链、NPC 是失忆的。核心动词「点外卖→看 NPC 演戏」是**消费循环**，不是**游玩循环**。要从「爽一次」长成「天天来」，还差几块关键拼图。

---

## 0. 玩法全景（它到底是什么）

读完代码后，当前「游戏」实际长这样：

```markdown
# Core Loop: 胡闹外卖（当前实际版本）

## Moment-to-Moment (0–30 秒)
- Action: 逛 15 家 NPC 店（首页分类/性格筛选 + 排序），点进店，点菜加购
- Feedback: CartBar 即时数字变化、Vant 组件动效
- Reward: 「我在点外卖了」的掌控感（零成本）

## Session Loop (5–30 分钟 · 实为 1 分钟 demo)
- Goal: 选地址 + 选备注 → 提交 → 看 NPC 状态机演出一整单
- Tension: ❌ 无。每单必赢，老板永远好笑，骑手永远到，饭永远香
- Resolution: 送达 → 弹彩蛋 → 5 星 + 戏精评价 + 老板回怼

## Long-Term Loop (hours–weeks)
- Progression: 12 个成就（普通/稀有/隐藏，纯计数解锁）
- Retention Hook: 胡闹动态（你自己的订单 + 随机 NPC 赞/评）+ 自埋点指标
```

**核心「决策点」只有一个，且设计得很好**：`Checkout.vue` 里的「地址（6 选）」和「备注（6 选）」不是装饰——它们是 **NPC 反应引擎的输入**（`config.json` 的 `addressTimeOffset` / `remarkTimeOffset` / `address_reactions` / `remark_reactions`，经 `npcEngine.ts` 注入话术变量）。这是全游戏唯一一处「玩家做了选择 → 世界真的不同」的地方。**这点要保住，别动。**

**「乐趣假说」**：NPC 戏精人格（5 种老板 × 5 种骑手）× 变量话术组合 × 零卡路里满足点外卖瘾 = 玩家笑 / 想截图。
- 首触交付？✅ 是的。引擎产出的是真的多样、贴品牌的台词。
- 持续？⚠️ 未验证 / 高危。除了新鲜感，没有任何重玩驱动力。

## 1. 亮点（真做得好的地方，证据在代码里）

| # | 亮点 | 证据 | 为什么是好的游戏设计 |
|---|------|------|------------------|
| 1 | **喜剧引擎是真正的产品**，且架构干净 | `npcEngine.ts` 变量注入（shopName/dishName/address/remark/riderName）+ 5 性格老板 × 5 原型骑手 × 备注/地址反应；`quotes.json` boss 20 / rider 20 / 各种 reactions；`reviewGenerator.ts` ~200 条模板分 11 类 | 这正中文档自己说的「灵魂在 NPC 戏精人格，不在技术栈」。组合爆炸给了首触的新鲜感 |
| 2 | **把结算页做成「输入」而非「确认」** | `Checkout.vue` 地址/备注选择 → `cookingTime.ts` 算时 + `npcEngine.ts` 分支话术 | 全游戏唯一有意义的玩家决策，且「选择改变演出」——这是核心动词真正的 decision |
| 3 | **节奏调速 + 生活映射是反无聊的神器** | `OrderDetail.vue` 默认 ×30 倍速 + `config.json` `demoSpeedOptions`；`getDemoLifestyle` 把秒数映射成「☕一杯咖啡的时间 / 📺追一集剧」 | 直接承认「等待是敌人」，把干等变成有梗的戏剧反差 |
| 4 | **新手引导又短又准** | `OnboardingGuide.vue` 3 屏 ≤30s，只答「是什么/怎么玩/彩蛋在哪」 | 符合我的 Onboarding Checklist：核心动词进场即懂、首屏有「哇」、末屏永远有开始按钮 |
| 5 | **截图/分享点存在** | `FeedPage.handleShare` 复制段子到剪贴板；`ReviewModal` 提交后庆祝态 | 命中品牌铁律「用户会截图发朋友圈吗」 |
| 6 | **工程卫生过硬** | `order.ts`/`achievement.ts` 全程 try/catch 兜底 localStorage；`sanitizeQuote` 防御脏数据；`metrics.ts` 7 事件自埋点；验收 114 项 89.5% 通过、0 真 bug | 对个人开发者极度重要——不会因为一个异常把 UI 搞崩 |

> 特别点名 **v13 修复**（`npcEngine.ts` 第 187–224 行）：用户没填备注时，老板不再无理由发火，改用中性接单语。这是**有玩家共情力的迭代**，说明团队在「NPC 该不该无缘无故骂用户」上做过正确的判断。保住这个判断。

## 2. 问题 / 风险（「玩法」在哪不合适）

### 🔴 P0 — 它是一台「模拟器」，不是「游戏」

- **没有失败、没有技巧、没有赌注。** 每一单都是保送赢。老板永远好笑、骑手永远到、饭永远香。没有 risk/reward、没有张力、除装饰外没有 agency。
- **四大痛点里有「想减肥管不住手」「无聊」**——这都是**习惯 / 复购**型问题。要解它们，产品需要**重复游玩**。但一台零赌注的玩具，新鲜感循环完 200 条模板后就哑火。**新鲜感衰减是头号威胁，而当前设计没有任何机制去对抗它。**
- 玩家视角三问之「他在做哪个决策？」→ 结算之后基本没有。看时间线自动推进是**被动消费**，不是游玩。

> 设计诊断：当前是一个 **reactive narrative toy（反应式叙事玩具）**，不是有 loop 能留人的 game。它在「首次接触」维度满分，在「留存」维度不及格。

### 🟡 P1 — 留存循环是**唯我论**（没有真正的社会）

- `FeedPage.vue` 用**随机** NPC 赞/评回应**你自己的**订单。没有另一个真人、没有竞争、没有共享梗经济。「传播」痛点（朋友圈/即刻）目前只靠手动复制一条段子。
- 成就（`achievements.json`）是**纯计数器**（order_count、same_shop×10…）——它们追踪**数量**，不是**技巧**也不是**有趣行为**。「回头客×3」奖励重复却不给任何**新能力**。这是最弱的一种进度（grind counter），很快会空。没有「改变你**怎么玩**」的元进度。

### 🟡 P1 — 选择不累积，世界不记得你

- v13 修复正确地让老板在你没备注时不再骂你。但代价是：**NPC 现在是个失忆症**——同一家店第 7 单，给的随机台词和第 1 单一模一样。
- 文档自己写的 AI 大脑愿景（「你点这家第 7 次了。要不我入股你胃吧」）才是**对的**留存点子——**NPC 持久记忆**——但它被推迟到 `whoknow-brain`，没进 MVP。没有它，重复访问体验完全相同。**这是「既定留存野心」与「已交付循环」之间最大的缺口。**

### 🟢 P2 — 决策空间窄且一次性

- 只有 2 个真输入（地址、备注），且每单独立。没有 build-up、没有「一局」、没有配装。对比已知爆款范式（韩国 FoodNeverComes，70 万单），其乐趣也在 NPC 惊喜，但**惊喜靠的是变化引擎**——本作目前是静态模板池，没有变化引擎。

### 🟢 P2 — 经济是「假的」（为痛点④ 正确），但也拿掉了所有长线动机杠杆

- 无货币、无解锁、无声望。对「零卡路里 / 不氪」完全正确，但也移除了游戏设计师用于长期动机的整个杠杆集。于是产品只能靠**内容新鲜感 + 社会 + 收藏**来激励——而当前只有收藏（成就），且很弱。

### 🟢 P2 — 指标已埋，但还没闭环成「设计问题」

- `metrics.ts` 把 7 个事件写进 localStorage，但文档里标着「数据指标 🔴 待聊透」。有仪表盘，没有「什么样算好 / 什么样算坏」的定义。对个人开发者 OK，但按我的规则：**playtest 前要先定义 broken 长什么样**——这一步还空着。

## 3. 平衡性评估

这里「平衡」不是公平，而是**节奏与喜剧 timing**。当前分布是**未调的随机**：

- 骑手迷路 20% 概率（`npcEngine.ts` `Math.random() < 0.2`）
- 有备注时老板发疯 35% 概率（`Math.random() < 0.35`）
- 时间全是 cosmetic（demo 倍速），没有「公平性」问题，只有「笑点密度」问题

**缺一张 tuning spreadsheet，也缺一个真正的 KPI**：每单目标「笑率 / 截图率」是多少？全 `[PLACEHOLDER]`。按我的规则，这是必须补的——不然你永远不知道循环「坏没坏」。

> 正向参考：`config.json` 把所有参数（出餐时间、倍率、偏移、chip 配色）抽到一处、业务代码零硬编码、预留 `_brainHooks`。**这是模范级的「改这里 = 改游戏性，无需改代码」设计**，为后续调平衡留了极好的口子。

## 4. 改进建议（按优先级，且都贴合你的约束：1 人开发 / ¥0 AI 预算 / 无后端）

### 🔴 P0（最高 ROI，直接补留存脊柱）
**1. 在 MVP-lite 里先上「NPC 记忆」，无需 AI。**
localStorage 记每家店的 `orderCount` + 上次备注；话术按 `orderCount >= N` 分支。这是**单次改动 ROI 最高**的一笔，且直接实现你们自己写下的愿景。成本极低（几天）。

### 🟡 P1
**2. 让成就「行为化」，别只「计数」。**
把 12 个成就从「下了 20 单」改成「用百慕大地址下单」（探索）、「连续 3 单不同老板」（多样性）、「把 5 种性格老板全惹毛过」（带精通的收藏）。给成就**意义**，并 nudges 重玩多样性。

**3. 把「胡闹动态」从「个人日志」变成「共享梗面」。**
哪怕全假：放几条来自全局池的「正在流行段子」；让用户「投稿」自己最胡闹的一单（先复制到共享 JSON/CDN，后期接）。给截图一个**目的地**。

### 🟢 P2
**4. 加一个轻量「戏精值 / 分数」每单。**
由选择算出（选百慕大 / 表演才艺 / 激怒暴躁老板 = 更高戏精值），显示成一个好玩的 meter，**不绑货币**。在不违反「不氪」前提下，给一个可追的**决策**和一个数字。

**5. 补 tuning 文档**：把「每单笑率 / 截图率」定义成真正的平衡 KPI，配套 spreadsheet。

**6. 保住不动的东西**：demo 调速、生活映射、新手引导、sanitize 防御、参数外置——这些都对了，别碰。

## 5. 下一步（我的头号建议）

**补一次真机 playtest，带真实用户，测两个指标：**
1. **笑率**（一单里玩家笑/截图了几回）
2. **次日 / 三日回访**（新鲜感衰减曲线到底多陡）

定义清楚「broken 长什么样」（比如：首触笑率 < 40% → 话术池不够；三日回访 < 15% → 缺留存脊柱），再决定 P0 那条 NPC 记忆要不要提前排进 M2。

## 6. 总结评分（设计视角，非工程）

| 维度 | 评分 | 说明 |
|------|:----:|------|
| 首触乐趣（喜剧引擎） | 🟢 9/10 | 真有灵魂，变量组合给力 |
| 核心决策设计 | 🟢 8/10 | 结算页做输入，是全作最亮的一笔 |
| 节奏 / 反无聊 | 🟢 8/10 | 调速 + 生活映射很聪明 |
| 新手引导 | 🟢 8/10 | 短、准、有哇点 |
| 留存循环 | 🔴 4/10 | 唯我、计数成就、NPC 失忆 |
| 社会 / 传播闭环 | 🟡 5/10 | 只有手动复制段子 |
| 长期「游戏性」脊柱 | 🔴 3/10 | 模拟器，非游戏；无赌注无技巧 |
| 工程可维护性 | 🟢 9/10 | 0 bug、防御齐全、参数外置 |

**总评**：一个**被严重低估的喜剧引擎**，包装在一个**长期设计偏薄的壳**里。方向对、灵魂对、首触对——只差把「一次性爽」焊成「天天来」的那几根梁。P0 那条 NPC 记忆，建议立刻排期。
